# Рабочий процесс для AI - Избежание зависаний

## Проблема

Команды типа `npm run build:admin`, `php artisan`, `git` часто зависают при выполнении через инструменты.

## Решения

### 1. Избегать выполнения долгих команд напрямую

**Вместо:**
```bash
npm run build:admin  # Может зависнуть
```

**Лучше:**
- Предоставлять инструкции пользователю
- Использовать готовые скрипты-обёртки
- Разбивать на маленькие шаги

### 2. Всегда использовать флаги для неинтерактивного режима

#### PHP Artisan:
```php
--no-interaction
--force
--quiet
```

#### Git:
```bash
--no-verify
--quiet
--no-edit
```

#### NPM:
```bash
--yes
--no-confirm
```

### 3. Использовать таймауты

Все команды должны иметь таймаут:
- npm: 300 секунд (5 минут)
- php artisan: 120 секунд (2 минуты)
- git: 60 секунд (1 минута)

### 4. Проверять состояние перед выполнением

Перед выполнением команды:
1. Проверить, не запущена ли уже
2. Проверить блокировки файлов
3. Проверить доступность ресурсов

## Рекомендуемый подход

### Для AI-ассистента:

1. **Предоставлять инструкции вместо выполнения**
   - Если команда может зависнуть, дать инструкцию пользователю
   - Пример: "Выполните: npm run build:admin"

2. **Использовать проверки перед выполнением**
   - Проверять наличие файлов
   - Проверять состояние процессов
   - Проверять доступность ресурсов

3. **Разбивать на маленькие шаги**
   - Вместо одной большой команды - несколько маленьких
   - Каждый шаг с проверкой результата

4. **Использовать готовые скрипты**
   - Создать скрипты для часто используемых операций
   - Скрипты с таймаутами и обработкой ошибок

## Альтернативный подход

Если команды часто зависают:

1. **Создать скрипты-обёртки** в `scripts/`
2. **Документировать шаги** в README
3. **Давать инструкции пользователю** вместо выполнения

Это гарантирует, что команды будут выполняться корректно.
