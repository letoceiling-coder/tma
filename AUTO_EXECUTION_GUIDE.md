# Руководство по автономному выполнению команд

## Проблема с зависаниями

При выполнении команд через инструменты могут возникать зависания из-за:
- Таймаутов
- Интерактивных запросов
- Долгого выполнения
- Блокировок файлов

## Решение: Использование инструкций вместо прямого выполнения

### Стратегия работы

1. **Для долгих команд (> 30 секунд):**
   - ✅ Предоставлять инструкцию пользователю
   - ✅ Не выполнять напрямую
   - ✅ Ждать подтверждения выполнения

2. **Для коротких команд (< 30 секунд):**
   - ✅ Выполнять с таймаутом
   - ✅ Добавлять флаги неинтерактивности
   - ✅ Проверять результат

3. **Для критических операций:**
   - ✅ Всегда давать инструкции
   - ✅ Показывать команду перед выполнением
   - ✅ Запрашивать подтверждение

## Примеры команд

### ✅ Правильно (инструкция):
```
Для сборки админ-панели выполните:
npm run build:admin

Команда может занять 1-3 минуты.
```

### ❌ Неправильно (прямое выполнение):
```bash
npm run build:admin  # Может зависнуть
```

## Рекомендации для AI

1. **Всегда проверять сложность команды** перед выполнением
2. **Использовать готовые скрипты** если они есть
3. **Давать инструкции** для команд сборки/деплоя
4. **Проверять файлы** перед чтением/редактированием

Это обеспечит стабильную работу без зависаний.
